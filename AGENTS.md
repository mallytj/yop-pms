## Project Structure

- **Backend**: Go with Chi router in `cmd/` and `internal/`
- **Frontend**: SvelteKit 5 with Runes in `web/`
- **Database**: PostgreSQL 18 with SQLC for type-safe queries
- **Cache**: Redis 7
- **API Contracts**: OpenAPI/Swagger schema-first approach

## Key Commands

### Development

```bash
make setup          # First-time setup (install tools, create .env, install npm deps)
make docker-up      # Start PostgreSQL, Redis, Adminer, Redis Commander, and server
make db-up          # Start only PostgreSQL + Redis (if running Go locally via Air)
make dev            # Start Go (Air hot-reload) + SvelteKit (Vite) concurrently
```

### Code Generation

```bash
make gen                # Full API contract generation (Swagger → OpenAPI → TypeScript types)
make swag               # Generate Swagger/OpenAPI specs from Go comments only
make sqlc               # Generate type-safe DB code from SQL queries
make gen-constraints    # Sync config/constraints.g.yml + web/src/lib/types/constraints.g.ts from live DB
```

Always run `make gen` after modifying Swagger comments in Go code. Pre-push hook
enforces. Run `make gen-constraints` after adding/changing DB check constraints
to keep generated YAML + TypeScript in sync.

### Testing

```bash
make test           # Run all tests (backend + frontend)
make test-backend   # go test -race ./...
```

### Quality

```bash
make audit    # Full quality suite: go mod verify, go vet, tests, govulncheck, svelte-check
make lint     # golangci-lint (Go) + Prettier validation (frontend)
make format   # go fmt (Go) + Prettier (frontend)
```

### Database

```bash
make reset-db        # Full DB teardown and restart (docker-compose down -v + up)
make goose-circle    # Complete Goose migration reset
```

## Architecture

### API Contract Flow (Schema-First)

1. Swagger annotations in Go handlers → `make swag` → `/api/swagger.json`
2. Convert to OpenAPI 3.0 → `/api/openapi.json`
3. Generate TypeScript types → `/web/src/lib/types/api.d.ts`

Frontend must not define own API types — all types come from generation.

### Backend (`cmd/` + `internal/`)

- Entry: `cmd/server/main.go` (config, DB pool, Redis, server init) +
  `cmd/server/api.go` (route registration)
- Domain logic: `internal/{booking,planner,pricing}/`
- Cross-cutting: `internal/platform/` (config, db, middleware, logging,
  apierror, cache, constraints, events, helpers, json, otel, validation, worker)
- Data access: `internal/store/` — **all generated by SQLC, never edit
  manually**
  - Write SQL in `internal/store/queries/`, then run `make sqlc`
- Always run `make lint` after generating code to catch any issues flagged by
  linters. Pre-push hook enforces linting and testing.

### Constraint Consistency System

DB check constraints synced to two generated files — **never edit manually**:

- `config/constraints.g.yml` — machine-readable constraint metadata (backend
  validation)
- `web/src/lib/types/constraints.g.ts` — TypeScript types (frontend validation)

Run `make gen-constraints` (requires live DB) to regenerate. See
`docs/guides/backend-constraints.md` and `docs/guides/frontend-constraints.md`.

### Frontend (`web/`)

- SvelteKit 5 with **Runes** (`$state`, `$derived`, `$effect` — not legacy
  stores where possible)
- Path aliases: `$lib`, `$components`, `$helpers`, `$stores`, `$types`,
  `$actions`
- API types from `$types` (generated `api.d.ts`)

### Database Conventions (enforced in migrations)

- **Financials:** `INTEGER` only (smallest currency unit — no floats)
- **Timestamps:** `TIMESTAMPTZ` exclusively
- **Primary keys:** UUIDv7 (time-sortable)
- **Foreign keys:** `RESTRICT` on delete (preserve history)
- **Soft deletes:** `deleted_at TIMESTAMPTZ` — uniqueness indexes use
  `WHERE (deleted_at IS NULL)`
- **Multi-tenancy:** Every tenant-isolated table has `property_id`; Row-Level
  Security enabled
- **Optimistic locking:** `version` column on high-concurrency tables
- **Text fields:** Check constraints on length; `CITEXT` for emails, usernames,
  codes
- Constraint naming: `{table}_{column}_{suffix}`

### Migrations

Goose in `/migrations/`. Zero-padded sequential filenames. Configure via
`GOOSE_*` env vars in `.env`.

## Local Services (docker-compose)

| Service         | Port | Purpose          |
| --------------- | ---- | ---------------- |
| PostgreSQL 18   | 5433 | Primary database |
| Redis 7         | 6379 | Cache / sessions |
| Adminer         | 8081 | DB admin UI      |
| Redis Commander | 8082 | Redis UI         |
| Go server       | 8080 | Backend API      |

Frontend dev: `http://localhost:5173` (Vite default). Swagger UI:
`http://localhost:8080/swagger/index.html`

## Key Decisions (see `/docs/adr/` for full context)

- **No ORM** — raw SQL via SQLC, type safety without abstraction overhead
- **Pure CSS** frontend — no CSS framework
- **`scratch`-based Docker image** — ~30MB, non-root user
- **Redis** transient state only (sessions, token blacklists, rate limiting) —
  not primary store

- **Constraints-driven validation**: Use
  `validation.Struct(input, "schema.table")` from
  `internal/platform/validation/` for struct validation against DB constraints.
  Add `constraints:"schema.table"` tag on nested slice fields.
