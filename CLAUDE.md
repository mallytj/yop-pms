# CLAUDE.md

Guidance for Claude Code working in this repo.

## Project Overview

**Yop PMS** — Property Management System. Monorepo, end-to-end type safety DB→frontend. Stack: Go (Chi) + SvelteKit 5 (Runes) + PostgreSQL + Redis. SQLC for type-safe DB access. OpenAPI as backend/frontend contract.

## Common Commands

All commands from repo root via `make`.

### Development

```bash
make setup       # First-time setup: install Go tools, create .env, install npm deps
make docker-up   # Start PostgreSQL, Redis, Adminer, Redis Commander, and server
make db-up       # Start only PostgreSQL + Redis (if running Go locally via Air)
make dev         # Start Go (Air hot-reload) + SvelteKit (Vite) concurrently
```

### Code Generation

```bash
make gen                # Full API contract generation: Swagger → OpenAPI → TypeScript types
make swag               # Generate Swagger/OpenAPI specs from Go comments only
make sqlc               # Generate type-safe DB code from SQL queries
make gen-constraints    # Sync config/constraints.g.yml + web/src/lib/types/constraints.g.ts from live DB
```

**Always run `make gen` after modifying Swagger comments in Go code.** Pre-push hook enforces this.
**Run `make gen-constraints` after adding/changing DB check constraints** to keep generated YAML + TypeScript in sync.

### Testing

```bash
make test          # Run all tests (backend + frontend)
make test-backend  # go test -race ./...
```

### Quality Checks

```bash
make audit    # Full quality suite: go mod verify, go vet, tests, govulncheck, svelte-check
make lint     # golangci-lint (Go) + Prettier validation (frontend)
make format   # go fmt (Go) + Prettier (frontend)
```

### Database

```bash
make reset-db       # Full DB teardown and restart (docker-compose down -v + up)
make goose-circle   # Complete Goose migration reset
```

## Architecture

### API Contract Flow (Schema-First)

1. Swagger annotations in Go handlers → `make swag` → `/api/swagger.json`
2. Convert to OpenAPI 3.0 → `/api/openapi.json`
3. Generate TypeScript types → `/web/src/lib/types/api.d.ts`

Frontend must not define own API types — comes from generation. Stale `api.d.ts` breaks TypeScript compilation.

### Backend (`cmd/` + `internal/`)

- Entry: `cmd/server/main.go` (config, DB pool, Redis, server init) + `cmd/server/api.go` (route registration)
- Domain logic: `internal/{booking,planner,pricing}/`
- Cross-cutting: `internal/platform/` (config, db, middleware, logging, apierror, cache, constraints, events, helpers, json, otel)
- Data access: `internal/store/` — **all generated by SQLC, never edit manually**
  - Write SQL in `internal/store/queries/`, then run `make sqlc`

### Frontend (`web/`)

- SvelteKit 5 with **Runes** (`$state`, `$derived`, `$effect` — not legacy stores where possible)
- Path aliases: `$lib`, `$components`, `$helpers`, `$stores`, `$types`, `$actions`
- API types from `$types` (generated `api.d.ts`)

### Constraint Consistency System

DB check constraints synced to two generated files — **never edit manually**:

- `config/constraints.g.yml` — machine-readable constraint metadata (backend validation)
- `web/src/lib/types/constraints.g.ts` — TypeScript types (frontend validation)

Run `make gen-constraints` (requires live DB) to regenerate. See the package
comment in `internal/platform/constraints/`.

### Database Conventions (enforced in migrations)

- **Financials:** `INTEGER` only (smallest currency unit — no floats)
- **Timestamps:** `TIMESTAMPTZ` exclusively
- **Primary keys:** UUIDv7 (time-sortable)
- **Foreign keys:** `RESTRICT` on delete (preserve history)
- **Soft deletes:** `deleted_at TIMESTAMPTZ` — uniqueness indexes use `WHERE (deleted_at IS NULL)`
- **Multi-tenancy:** Every tenant-isolated table has `property_id`; Row-Level Security enabled
- **Optimistic locking:** `version` column on high-concurrency tables
- **Text fields:** Check constraints on length; `CITEXT` for emails, usernames, codes
- Constraint naming: `{table}_{column}_{suffix}`

### Migrations

Goose in `/migrations/`. Zero-padded sequential filenames. Configure via `GOOSE_*` env vars in `.env`.

## Key Decisions (see `/docs/adr/` for full context)

- **No ORM** — raw SQL via SQLC, type safety without abstraction overhead
- **Pure CSS** frontend — no CSS framework
- **`scratch`-based Docker image** — ~30MB, non-root user
- **Redis** transient state only (sessions, token blacklists, rate limiting) — not primary store
